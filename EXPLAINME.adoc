[quote, README]
____
ZeroProb.jl is a Julia library for handling zero-probability events in continuous
probability spaces. It provides rigorous mathematical tools for reasoning about
events with measure zero, including density-based relevance, Hausdorff measures,
paradox demonstrations, tail risk modelling, and black swan event analysis.
In continuous probability distributions, events with measure zero can still occur.
This library provides alternative relevance measures, classical paradox
demonstrations, and application-oriented tools for extreme events.
____

== What It Actually Does Today

=== Claim: Three complementary relevance measures for zero-probability events

*True.* The core of `src/measures.jl` implements:

- `density_ratio(dist, x, baseline)` — divides the PDF value at `x` by the PDF
  at `baseline`; a Normal(0,1) mean vs. 3σ gives ratio ≈ 228 as shown in the README.
- `hausdorff_measure(set, dimension)` — computes the Hausdorff measure at a given
  fractal dimension; `hausdorff_dimension(set)` estimates box-counting dimension.
  Verified: `construct_cantor_set(iterations=10)` has Lebesgue measure 0 but
  non-zero Hausdorff measure at dimension log(2)/log(3) ≈ 0.631.
- `epsilon_neighborhood_prob(dist, x, eps)` — wraps `Distributions.cdf` to compute
  P(|X - x| < eps); `estimate_convergence_rate(eps_values, probs)` fits the
  convergence exponent.

Additional information-theoretic measures in `src/measures.jl`:
`radon_nikodym_derivative`, `total_variation_distance`, `kl_divergence`,
`fisher_information`, `entropy_contribution`, `conditional_density`,
`almost_surely`, `measure_zero_test`.

=== Claim: Classical paradox demonstrations (11 total)

*All implemented* in `src/paradoxes.jl`:

`continuum_paradox`, `borel_kolmogorov_paradox`, `rational_points_paradox`,
`uncountable_union_paradox`, `almost_sure_vs_sure`, `construct_cantor_set`,
`banach_tarski_paradox`, `vitali_set_paradox`, `gabriels_horn_paradox`,
`bertrand_paradox`, `buffon_needle_problem`.

These are pedagogical functions — they print explanations and return the
key mathematical objects (Cantor set as a nested interval structure, Vitali set
as a coset representative set, etc.). They are not proofs; they are concrete
computational demonstrations.

=== Claim: Black swan and extreme event applications

`src/applications.jl` implements `BlackSwanEvent`, `MarketCrashEvent`,
`BettingEdgeCase` types with `impact_severity`, `expected_impact`, `expected_value`,
`handles_black_swan`, `handles_zero_prob_events`, `handles_zero_prob_event`.
These are designed for use in `@ensure` assertions within Axiom.jl's verification
DSL, as shown in the `src/ZeroProb.jl` module docstring example.

=== Claim: Integration with ECHIDNA probabilistic reasoning

*Architecture described; not wired.* The README section "Integration with ECHIDNA"
describes `density_ratio` for premise selection, `epsilon_neighborhood_prob` for
proof search guidance, and Hausdorff measures for anomaly detection. The
ZeroProb types are designed to support these integration points, but the
ECHIDNA integration is not yet implemented.

*Honest caveat:* The test suite (280 tests per README, split across 8 test files)
exercises the mathematical functions directly. The visualisation functions
(`src/visualization.jl`) require `Plots.jl` which is a test-time optional
dependency; they are excluded from the default test run if Plots is unavailable.
The GPU/coprocessor backend extensions follow the same AcceleratorGate dispatch
pattern as QuantumCircuit.jl — interface-correct, hardware paths conditional.

== How It Works

Critical path for density-ratio relevance:

. Construct a `ContinuousZeroProbEvent(dist, value)` wrapping a `Distributions.jl`
  distribution and a specific outcome.
. Call `probability(event)` — returns 0.0 (correct for continuous distributions).
. Call `relevance(event)` — returns the PDF value at the event point (non-zero).
. Call `density_ratio(dist, x1, x2)` for comparative analysis.

For fractal analysis:
. Call `construct_cantor_set(iterations=n)` — builds nested interval list.
. Call `hausdorff_dimension(set)` — box-counting estimate.
. Call `hausdorff_measure(set, dimension)` — measure at that dimension.

  `@ensure handles_zero_prob_events(output, MarketCrashEvent())`. The

  formal verification layer.
